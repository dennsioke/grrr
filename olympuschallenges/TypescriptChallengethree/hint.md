CRITICAL: The useParallax hook must preserve its existing behavior exactly when keyframes are not provided. Only when the keyframes array is present and non-empty should you set up the onProgressChange callback and modify the element config. If you unconditionally inject onProgressChange or alter how parallax element props are constructed, you will break every existing test (52+ baseline tests). Guard all keyframe-related logic in useParallax behind an `if (keyframes && keyframes.length > 0)` check — the non-keyframe code path must remain completely untouched.

When using parseValueAndUnit from parallax-controller to parse effect values, be aware that its default unit is '%'. This is correct for translate effects but wrong for rotate and scale. Rotate values must use 'deg' as the default unit and scale values must be unitless. Pass the correct default unit for each effect type when calling parseValueAndUnit — do not rely on its built-in default. The description states the output units explicitly: translateX/translateY use '%' for numeric values, rotate uses 'deg', and scale/scaleX are unitless.

Unit consistency validation across keyframe stops must compare the actual parsed CSS unit strings (e.g., 'px' vs '%'), not JavaScript typeof checks. Two stops like `translateY: '10px'` and `translateY: '50%'` must throw because 'px' !== '%', which requires parsing both values and comparing their unit strings. A typeof check would see both as 'string' and wrongly pass.

Do not regenerate package-lock.json or run npm install with flags that rewrite the lockfile. The Dockerfile handles dependency installation.

Why this hint exists:
Every agent run that attempted this problem either (a) broke all baseline tests by unconditionally modifying the useParallax hook's code path even for non-keyframe usage, or (b) got rotate/scale units wrong because parseValueAndUnit defaults to '%'. Both are systematic blind spots — the description says "when keyframes are present" but agents consistently fail to conditionalize, and the per-effect unit mapping requires explicit default-unit overrides that agents skip. These hints make the conditional guard and unit-mapping requirements explicit without revealing architecture.
